**Daisy Shang, NullTemp · 2026**

## 第一阶段：项目构思与调研（第 1 天）

### 1.1 初始构想

本项目源于将机器学习应用于时域天文学的兴趣。核心问题非常直接：给定一条光变曲线（即亮度随时间变化的观测序列），能否自动识别产生该光变曲线的天体类型？

经过调研，我们选择 PLAsTiCC（Photometric LSST Astronomical Time-series Classification Challenge，LSST 测光时域分类挑战赛）作为数据集。该数据集包含 18 类天文瞬变源与变源的模拟光变曲线，旨在模拟 Vera C. Rubin 天文台 LSST 巡天项目未来将产生的观测数据。数据集公开可用且带有标注，非常适合监督学习任务。

### 1.2 技术路线选择

我们评估了以下方案：

| 方案 | 优点 | 缺点 | 决策 |
|------|------|------|------|
| CNN 处理光变曲线图像 | 文献中有成功先例（BTSbot、SCONE） | 需要大量 GPU 算力和更多数据 | 搁置 |
| LSTM 处理原始序列 | 天然适合时序建模 | 需要大规模训练集 | 已测试，效果不佳 |
| 特征提取 + XGBoost | 速度快、可解释性强、擅长表格数据 | 需要领域知识设计特征 | **选定为主要方案** |

选择特征提取路线的理由是：手工设计的特征编码了天文学领域知识，而神经网络需要数百万样本才能从零学到这些知识。在仅有约 7,000 个训练样本的条件下，梯度提升树提供了最优的取舍。

### 1.3 工具与环境

- Python 3.11、XGBoost、scikit-learn、PyTorch、Gradio
- PLAsTiCC 数据集（通过公开镜像获取）

## 第二阶段：数据管道搭建（第 1–2 天）

### 2.1 数据集获取

PLAsTiCC 数据集以 Parquet 格式下载。初次尝试加载全部 85 个测试分区文件导致内存溢出。解决方案是仅加载训练与验证分区（各一个文件，分别约 7,000 和 1,500 个天体）。

### 2.2 数据格式问题

Parquet 文件使用嵌套的对象数组格式：

- `times_wv`：一维数组，每个元素为 `[MJD, 波段波长]`
- `lightcurve`：一维数组，每个元素为 `[流量, 流量误差]`

这要求在数据处理前将对象数组转换为二维数值数组。最初代码假设的数据格式为 CSV 扁平列，这是第一个主要的调试障碍。

其他遇到的问题：
- 波段值以波长（埃）表示（如 6222 对应 r 波段），而非最初假设的 0–5 索引
- 一次性加载所有分区耗尽了系统内存（总计约 12 GB）

### 2.3 特征工程

特征设计经历了三次迭代：

**V1（20 个特征）**——仅基础统计量：
- 时间跨度、观测点数、流量的均值/标准差/中位数/最小值/最大值/极差
- 偏度、峰度、信噪比（均值与最大值）
- 振幅（P5–P95 范围）、超出 1σ 的比例
- 最大上升/下降速率、峰值位置、自相关

**V2（41 个特征）**——增加分波段和颜色信息：
- 六个波段（u, g, r, i, z, y）各自的均值、标准差和信噪比（18 个特征）
- 色指数：u−g, g−r, r−i, i−z, z−y（5 个特征）
- 这些特征捕捉了光谱能量分布，不同类别之间存在显著差异

**V3（47 个特征）**——增加稳健统计量与宿主星系信息：
- 四分位距（IQR）、中位数绝对偏差（MAD）
- 红移、宿主星系光谱红移与测光红移
- 观测到的独特波段数量

最终的 47 特征集用于所有生产模型。

## 第三阶段：模型训练（第 2–3 天）

### 3.1 基线模型（V1）

- **模型**：XGBoost，默认参数
- **训练集**：3,080 个天体（每类 200 个，仅 r 波段）
- **特征**：20 个统计特征
- **结果**：准确率 54.3%，宏观 F1 51.6%

低分符合预期——20 个简单特征和有限的训练数据不足以完成 14 类分类任务。

### 3.2 改进基线（V2）

- **训练集**：5,910 个天体（扩大采样，全部波段）
- **特征**：41 个特征
- **结果**：准确率 68.6%，宏观 F1 59.0%（+14.3 个百分点）

提升来自两个变化：更多的训练数据和分波段颜色特征。后者在区分超新星类型和变星方面尤其有效。

### 3.3 优化模型（V3）

- **训练集**：7,335 个天体（合并训练集与验证集）
- **特征**：47 个特征，包含缺失值处理
- **超参调优**：参数网格 3 折交叉验证，类别加权损失函数
- **结果**：准确率 82.8%，宏观 F1 76.8%

关键优化决策：
- `max_depth=10, learning_rate=0.02, n_estimators=500`
- 类别加权处理样本不平衡（如 Mira 变星仅 33 个样本，Ia 型超新星 2,544 个）
- L2 正则化（`reg_lambda=1.5`）防止过拟合

### 3.4 超参数搜索（Optuna）

使用 3 折交叉验证对 11 个超参数进行了 80 次自动搜索。搜索策略与参数范围的确定借助了 AI 工具进行辅助设计。结果显示手动调参已接近最优，准确率为 82.2%（对比手动 82.8%）。微小的差距表明模型在此特征集上已接近性能上限。

## 第四阶段：深度学习实验（第 3–4 天）

### 4.1 实验动机

有了 RTX 4060 Ti 显卡，我们希望测试深度学习能否超越特征工程方案。共测试了五种架构：

| 模型 | 输入 | 准确率 | 备注 |
|------|------|--------|------|
| BiLSTM + Attention | 原始光变曲线 (200×4) | 27.0% | 87 轮，GPU；未收敛 |
| Transformer 编码器 | 原始光变曲线 (200×5) | 26.0% | 86 轮，GPU；类似失败 |
| 2D 热力图 CNN | 时间×波段图像 (48×6×3) | 33.5% | 102 轮，GPU；轻微改善 |
| MLP | 47 个手工特征 | 69.2% | 82 轮，GPU；尚可但不及 XGBoost |
| 对比学习预训练（SimCLR） | 原始光变曲线 → 学习嵌入 | 34.4% | 150 轮，15 秒；嵌入不够有区分力 |

### 4.2 分析

所有基于原始光变曲线训练的深度学习模型均严重欠拟合。首要原因是数据量不足：约 4,000 条训练序列不足以让深层网络从嘈杂、异质的光变曲线中学习到有意义的时序表示。手工设计的特征隐式编码了几十年的天文学领域知识（例如色指数作为温度代理、振幅比作为爆发能量指标）。

对比学习实验证实了这一判断。虽然 SimCLR 编码器快速收敛（损失在 GPU 上 15 秒内从 3.2 降至 0.12），但学到的嵌入用于分类时仅达到 34.4% 的准确率。若有百万级样本（如 Zwicky Transient Facility 巡天数据），该方法很可能具备竞争力。

### 4.3 GPU 效用评估

使用 LightGBM 的 GPU 后端（`device='gpu'`）测试了 GPU 加速效果。对于我们的数据规模（约 8,000 样本 × 47 特征），GPU 训练反而比 CPU 更慢（每次试验 7.3 秒 vs. 1.1 秒）。数据传输开销超过了计算收益。GPU 加速仅对显著更大的数据集才有意义。

## 第五阶段：图形用户界面（第 4–5 天）

### 5.1 框架选择

选用 Gradio 构建界面，因为它对机器学习演示有良好的原生支持，且搭建简单。界面包含三个标签页：

1. **Upload CSV**：用户上传自己的光变曲线数据进行分类
2. **Demo**：随机抽取 PLAsTiCC 天体，展示预测结果与真实标签的对比
3. **Benchmark**：评估模型在指定数量测试样本上的准确率

### 5.2 国际化

实现了中英双语界面。标签页名称使用静态双语文本；Markdown 内容和组件标签通过 `gr.update()` 动态切换。按钮和下载组件因 Gradio 6 的 API 限制使用静态双语标签。

### 5.3 跨平台启动器

开发了基于 Python 的启动器（`launcher.py`），实现以下功能：

- 自动安装依赖（pip）
- PLAsTiCC 数据集下载提示（约 35 MB，需用户确认）
- 端口冲突解决（扫描 38000–38020）
- 服务器就绪检测（HTTP 轮询，就绪后再打开浏览器）
- 启动时清理旧进程

启动器在 Windows、macOS 和 Linux 上行为一致。

### 5.4 分发方案

准备了两个分发包：

- `deliver/`——中英双语版
- `deliver_en/`——纯英文版

两者均包含完整的应用程序、模型文件和源代码。用户仅需 Python 3.10+。

## 第六阶段：验证与结果

### 6.1 最终模型性能

在 1,295 个天体的保留测试集上评估，涵盖 14 个类别：

| 指标 | 值 |
|------|-----|
| 准确率 | 81.93% |
| 宏观平均 F1 | 75.95% |
| 加权 F1 | 81.59% |

### 6.2 各类别性能

| 类别 | F1 分数 |
|------|---------|
| 食双星 | 0.987 |
| M 矮星 | 0.985 |
| 活动星系核 | 0.984 |
| 天琴座 RR 型变星 | 0.963 |
| 超亮超新星（SLSN-I） | 0.915 |
| 微透镜单星 | 0.902 |
| Ia 型超新星 | 0.868 |
| 潮汐瓦解事件 | 0.759 |
| II 型超新星 | 0.663 |
| 千新星 | 0.600 |
| Ibc 型超新星 | 0.579 |
| 特殊 Ia 型超新星（91bg 类） | 0.438 |
| 特殊 Ia 型超新星（SNIax） | 0.360 |

表现最好的类别（食双星、M 矮星、活动星系核）具有高度独特的周期性光变曲线。表现最差的是具有相似光度演化特征的超新星亚型，即使是专业天文学家在没有光谱的情况下也难以区分它们。

### 6.3 混淆分析

最常见的误分类：

- II 型超新星 ↔ Ia 型超新星（1,295 个样本中 61 次相互混淆）
- Ibc 型超新星 ↔ II 型超新星
- 特殊 Ia 亚型 ↔ 普通 Ia 型

这些混淆在物理上是有意义的——不同核心坍缩和热核爆发机制的超新星，其宽波段光变曲线形态存在显著重叠。明确分类通常需要光谱后续观测。

## 第七阶段：经验与教训

1. **特征工程比模型架构更重要**。对于小型表格数据集，47 个精心设计的特征以大幅优势超越了所有深度学习方法。

2. **数据量是天文学中深度学习的瓶颈，而非算力**。仅有约 7,000 个训练样本，即使是性能不错的 GPU（RTX 4060 Ti）也无法挽救表现不佳的神经网络。

3. **梯度提升树在成百上千样本的表格数据上仍是 state-of-the-art**。XGBoost 在我们的实验中持续优于 MLP、LSTM、Transformer 和 CNN。

4. **GUI 开发需要与模型开发同样的精力**。处理跨平台兼容性、依赖管理、数据缺失场景和国际化占用了大量时间。

5. **领域知识不可替代**。色指数和宿主星系红移等特征——对天文学家显而易见——若仅依靠神经网络从数据中自行发现，需要数量级更大的训练集。
